Part 1. Loading The Input Dataset

Dataset source, historical background, and why this corpus is suitable for Spam/Ham classification.

Dataset Overview

The dataset used in this project is sourced from the GitHub archive: enron_spam_data . The notebook describes this version as a pre-processed and consolidated CSV file, while the original dataset consisted of many individual text files.

Historical context: Enron went bankrupt in 2001 after a major financial fraud scandal. During the investigation, the U.S. Federal Energy Regulatory Commission seized and publicly released Enron's email server database.
Classification context: Enron's released server mainly contained work emails. In 2006, V. Metsis, I. Androutsopoulos, and G. Paliouras injected real spam collected from spam traps, SpamAssassin, and web sources to create six balanced Enron Ham and Spam datasets.

The notebook frames this dataset as a practical Spam/Ham classification corpus because it combines authentic corporate Ham emails with externally collected Spam examples.

Part 3. Missing Data Analysis

Missing Subject and Message patterns, handling decisions, and subject-line behavior signals.

Missing Data Analysis

The EDA first checks missing values across the raw dataset, then separates missingness into Subject-only, Message-only, and both-feature cases. The notebook makes different decisions for the two text fields because an email body and an email subject do not carry the same modeling value.

Message rule: if Message is null or an empty string, drop the record because an email without body content has no predictive value.
Subject rule: if Subject is null or an empty string, keep the record and impute it with No Subject.
Pattern checks: count both-feature missingness, inspect Spam/Ham distribution among missing-subject records, and calculate missing-subject rates within each class.

Spam vs Ham Language

Spam vocabulary claim: Spam is promotional and deceptive, with words like click, now, make, money, product, and want. The notebook interprets this as urgency-driven phishing or unsolicited marketing.
Ham vocabulary claim: Ham reflects Enron's business operations, with words such as deal, billion, price, issue, work, let, know, and contact.

Spam contains investment and legal disclaimer language, suggesting pump-and-dump style campaigns. Ham contains workplace phrases and Enron-specific names/entities, which the notebook flags as dataset bias. The EDA specifically warns that names such as Vince Kaminski and organization tokens such as hou, ect, and corp should be treated carefully as custom stopwords to reduce memorization.

The notebook also states that URL patterns such as http www should be stripped with Regex, while financial terminology should be retained because it is a critical signal for malicious emails.

Email Length By Label

Classification value: the EDA concludes that Message_Length and Punct_Count should be retained for later model training.

The notebook rejects the assumption that Spam is always excessively long. It describes Spam as concise and template-like, while Ham can vary from a one-word reply to very long forwarded chains, reports, contracts, or handbook-like text.

The EDA chooses to prune the sub-10% long-tail outlier group while preserving enough data and avoiding severe class imbalance. The notebook justifies the 714.5-word threshold as practical because ordinary business or promotional emails rarely exceed roughly two A4 pages.

The notebook's pruning claim is that texts above this threshold are mostly structural artifacts, deeply nested forward chains, attached reports dumped as text, or raw HTML junk. Forcing the model to learn from them would add noise and increase overfitting risk.

Final EDA Takeaways

Balanced labels: Spam and Ham are close enough for a stable classification starting point.
Text cleanup needed: Enron-specific words, names, URLs, and email artifacts can dominate the vocabulary.
Feature signals: missing subject, uppercase subject behavior, length, punctuation, and financial bigrams are useful classification clues.
